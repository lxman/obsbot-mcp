# v0.4.0 — Multi-camera + Tool Renaming Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.4.0 — support up to four OBSBOT Tiny 2 cameras identified by serial, and rename the 30-tool surface to a consistent 34-tool surface — as one breaking change.

**Architecture:** One helper process per camera (`HelperProcess` already has no shared state). `DeviceManager` becomes a `Map<serial, entry>` registry that binds cameras by serial, reading the serial over the framed-V3 GET path (`frame[1] = 0x01`). Tools gain an optional `camera` selector. The tool surface is rewritten once — new name + selector + any split — to avoid touching each tool twice.

**Tech Stack:** TypeScript (ESM, strict), Zod schemas, Vitest. Native helper (`helper.m`) already matches cameras by `locationID` and reports it from `enumerate` — no further native work in this plan.

## Global Constraints

- **Hard rename, no aliases.** Old tool names must NOT resolve. (naming spec §1)
- **Single-camera must not regress.** One camera → caller never passes `camera`, never sees a serial. (multi-camera spec §5)
- **Serial is identity; `locationID` is only a handle to open with.** Never persist a `locationID→camera` binding as truth. (multi-camera spec §2)
- **`frame[1]` stays `0x25` for SETs.** `0x01` is the header-only GET flavour ONLY. Never flip the constant globally. (tiny2_specification.md §4.2)
- **Reply validation:** a framed GET reply is trusted only if magic + header CRC + payload CRC parse AND `cmd` and `seq` match the request. The selector-2 mailbox retains the previous reply. (tiny2_specification.md §4.3)
- **Serial:** `UG_GET_SN`, wireCmd `0x18C8`, receiver `0x0D`, 14 ASCII bytes. (tiny2_specification.md §4.4)
- **Target version:** `package.json` → `0.4.0`. CHANGELOG entry flagged BREAKING.
- Run the full suite with `npx vitest run`; typecheck with `npx tsc -p tsconfig.json --noEmit`.

```ts
// src/codec/frame.ts
export interface FrameOpts { seq: number; cmd: number; receiver: number; payload: Buffer; sender?: number; flags?: number; }

export function buildFrame(o: FrameOpts): Buffer {
  const frame = Buffer.alloc(60);
  frame[0] = 0xaa;
  frame[1] = o.flags ?? 0x25;                 // 0x25 = SET (nested payload); 0x01 = header-only GET
  u16le(o.seq).copy(frame, 2);
  u16le(12).copy(frame, 4);
  frame[8] = o.sender ?? 0x0a;
  frame[9] = o.receiver;
  u16le(o.cmd).copy(frame, 10);
  u16le(crc16usb(frame.subarray(0, 12))).copy(frame, 6);   // header CRC recomputed AFTER fields set
  if (o.payload.length > 0) {
    const len2 = o.payload.length;
    u16le(len2).copy(frame, 12);
    o.payload.copy(frame, 16);
    const seg = frame.subarray(12, 12 + len2 + 4);
    u16le(crc16usb(seg)).copy(frame, 14);
  }
  return frame;
}
```

```ts
// src/codec/commands.ts
import { OP_BY_NAME } from "./opcodes.js";
// VendorFrame already wraps buildFrame; extend the wrapper to carry flags.
export const encodeVendorGet = (name: string): VendorFrame => {
  const op = OP_BY_NAME.get(name);
  if (!op || op.wireCmd == null) throw new Error(`unknown or non-sendable opcode: ${name}`);
  return vendorFrame({ cmd: op.wireCmd, receiver: op.receiver, payload: Buffer.alloc(0), flags: 0x01 });
};
```

```ts
// Add to ObsbotTransport (transport.ts): readSerial(): Promise<string>;
// Implement identically in macos/linux/windows transports (or a shared helper):
async readSerial(): Promise<string> {
  const seq = this.nextSeq();
  const req = encodeVendorGet("UG_GET_SN").buildFrame(seq);
  await this.xuRaw(VENDOR_XU_SELECTOR, req);           // SET_CUR selector 2
  const wantCmd = 0x18c8;
  for (let i = 0; i < 8; i++) {                        // poll: reply latency varies, mailbox is stale-prone
    const raw = await this.xuGetRaw(VENDOR_XU_SELECTOR, 60);
    try {
      const f = parseFrame(raw);                       // validates magic + header + payload CRC
      if (f.cmd === wantCmd && f.seq === seq && f.payload.length > 0) return decodeSerial(f.payload);
    } catch { /* not our reply yet */ }
  }
  throw new Error("readSerial: no valid UG_GET_SN reply");
}
```
